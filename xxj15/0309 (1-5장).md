## 📍 2장 : 자바스크립트란?

- Node.js

- JS
  - HTML, CSS와 함께 웹을 구성하는 요소 중 하나로 **웹 브라우저에서 동작하는 유일한 프로그래밍 언어**

## 📄 인터프리터 언어 vs. 컴파일러 언어

| 비교 항목     | 컴파일러 언어                                                                                            | 인터프리터 언어                                                                                                           |
| :------------ | :------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------ |
| **동작 방식** | 코드가 실행되기 전 단계인 **컴파일 타임**에 소스코드 전체를 한번에 **머신 코드**로 변환한 후 실행한다.   | 코드가 실행되는 단계인 **런타임**에 문 단위로 한 줄씩 중간 코드(intermediate code)인 **바이트코드**로 변환한 후 실행한다. |
| **실행 파일** | 실행 파일을 생성한다.                                                                                    | 실행 파일을 생성하지 않는다.                                                                                              |
| **단계 구분** | 컴파일 단계와 실행 단계가 분리되어 있다. 명시적인 컴파일 단계를 거치고, 명시적으로 실행 파일을 실행한다. | 인터프리트 단계와 실행 단계가 분리되어 있지 않다. 인터프리터는 한 줄씩 바이트코드로 변환하고 즉시 실행한다.               |
| **수행 횟수** | 실행에 앞서 컴파일은 **단 한 번** 수행된다.                                                              | 코드가 실행될 때마다 인터프리트 과정이 반복 수행된다.                                                                     |
| **실행 속도** | 컴파일과 실행 단계가 분리되어 있으므로 코드 실행 속도가 **빠르다**.                                      | 인터프리트 단계와 실행 단계가 분리되어 있지 않고 반복 수행되므로 코드 실행 속도가 **비교적 느리다**.                      |

## 📍 3장 : 자바스크립트 개발 환경과 실행 방법

### JS 실행 환경

- 자바스크립트는 **브라우저와 Node.js** 라는 두 가지 주요 환경에서 실행될 수 있으며, 각각의 목적과 제공하는 기능에 차이가 있다.

- 공통점: 둘 다 자바스크립트 엔진을 내장하고 있어 **ECMAScript(JS 코어 성분)**를 해석하고 실행할 수 있다. 기본적인 자바스크립트 문법은 두 환경에서 동일하게 동작한다.
- 차이점:
  - 브라우저 (Browser) : 주 목적이 웹페이지를 화면에 렌더링하는 것.
    - DOM API 제공 (HTML 요소 조작)
  - Node.js : 주 목적이 브라우저 외부에서 실행 환경을 제공하는 것
    - DOM API 미제공 (직접적인 HTML 조작 불필요)

### Node.js

- **npm** : node package manager - Node.js에서 사용할 수 있는 모듈들을 패키지화해서 모아둔 저장소 역할과 패키지 설치 및 관리를 위한 CLI를 제공한다.

## 📍 4장 : 변수

- **변수** : 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름
  - 즉, 변수는 **값의 위치를 가리키는 상징적인 이름** 이라고 생각하면 된다.

  - 변수에 값을 저장하는 것을 **할당(대입, 저장)** 이라 하고, 변수에 저장된 값을 읽어 들이는 것을 **참조**라고 한다.

- **식별자** = 변수 이름 (어떤 값을 구별해서 식별할 수 있는 고유한 이름)
  - 값이 아닌 **메모리 주소**를 기억함

- **변수 선언** : var, let, const 키워드 (var 키워드의 여러 단점을 보완하기 위해 let, const 키워드가 도입됨)

1. var 키워드

```js
var score;
```

- var 키워드를 사용한 변수 선언은 선언 단계와 초기화 단계가 동시에 진행됨.
  - 어떠한 값도 할당하지 않아도 암묵적으로 *undefined*로 초기화가 자동 수행됨
- **ReferenceError** : 변수 뿐 아니라 함수, 클래스 등 모든 식별자를 사용하려면 반드시 선언이 필요하다. 선언하지 않은 식별자에 접근하면 ReferenceError(참조 에러)가 발생.

### 변수 선언의 실행 시점과 변수 호이스팅

```js
console.log(score);

var score;
```

- JS 코드는 인터프리터에 의해 한 줄씩 순차적으로 실행되므로 `console.log(score)`가 먼저 실행되고 순차적으로 다음 코드가 실행된다. 따라서 위의 코드는 ReferenceError가 발생할 것으로 보이지만, 참조 에러가 발생하지 않고 **undefined**가 출력된다.

  ✔️ 이유 : 변수 선언이 소스 코드가 한 줄씩 순차적으로 실행되는 시점(런타임)이 아니라 그 이전 단계에서 먼저 실행되기 때문이다.
  - 그 이전 단계 = 소스코드 실행을 위한 준비 단계인 소스코드 평가 과정
  - JS 엔진은 소스코드의 어디에 있든 상관없이 **변수 선언**을 다른 코드보다 먼저 실행함.

💡 이것을 **변수 호이스팅**이라고 함. (JS의 고유한 특징)

### 값의 할당

```js
var score = 90;
```

⚡️ **변수 선언과 값의 할당의 실행 시점은 다르다.**

- 변수 선언 : 런타임 이전에 먼저 실행됨
- 값의 할당 : 소스코드가 순차적으로 실행되는 시점인 런타임에 실행됨.

```js
console.log(score); // undefined
var score = 90;
console.log(score); // 90
```

### 값의 재할당

- 재할당 : 이미 값이 할당되어 있는 변수에 새로운 값을 또다시 할당하는 것
  - 변수는 재할당 가능(`var`) / 상수는 재할당 불가 (`const`)

### 변수 네이밍 규칙

- 대표적인 네이밍 컨벤션

```js
var firstName; // 카멜 케이스
var first_name; // 스네이크 케이스
var FirstName; // 파스칼 케이스

// 헝가리언 케이스
var strFirstName; // type + identifier
var $elem = document.getElementById("myId"); // DOM 노드
var observable$ = fromEvent(document, "click"); // RxJS옵저버블
```

- JS에서 일반적으로 변수나 함수의 이름에는 카멜케이스를 사용하고
- 생성자 함수, 클래스 이름에는 파스칼 케이스를 사용함

## 📍 5장 : 표현식과 문

### 값과 리터럴

- 값 : 식(표현식)이 평가되어 생성된 결과

- 리터럴 : 사람이 이해할 수 있는 문자 또는 약속된 기호를 사용해 값을 생성하는 표기법

```js
100 // 정수 리터럴
10.5 // 부동 소수점 리터럴
null // null 리터럴
[ 1, 2, 3 ] // 배열 리터럴
function() {} // 함수 리터럴
```

### 표현식

- **표현식** : 값으로 평가될 수 있는 문(statement) -> 표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조함.

```js
var score = 100;
```

-> 100 (리터럴)도 표현식이다

### 문(statement)

- statement : 프로그램을 구성하는 기본 단위이자 최소 실행 단위
  - 여러 토큰으로 구성되는데, 토큰은 **문법적인 의미를 가지며, 문법적으로 더 이상 나눌 수 업슨 코드의 기본 요소**를 의미함
  - 선언뭄ㄴ, 할당문, 조건문, 반복문 등

```js
var sum = 1 + 2;
```

- var, sum, =, 1, +, 2, ;은 모두 각각 토큰임

### 세미콜론

- 세미콜론은 문의 종료를 나타낸다. 즉, JS엔진은 세미콜론으로 문이 종료한 위치를 파악하고 순차적으로 하나의 문을 실행한다.
  - 문을 끝낼 때는 세미콜론을 붙여야 하는데, JS에서는 세미콜론을 자동으로 붙여주는 세미콜론 자동 삽입 기능이 암묵적으로 수행되기 때문에 세미콜론은 생략 가능하다.

### 표현식인 문과 표현식이 아닌 문

```js
var x; // 변수 선언문은 값으로 평가될 수 없으므로 표현식이 아님

x = 1 + 2; // 1, 2, 1 + 2, x = 1 + 2는 모두 표현식.
// x = 1 + 2는 표현식이면서 완전한 문이기도 하다
```

⚡️ 표현식인 문과 표현식이 아닌 문을 구별하는 가장 명료한 방법은 변수에 할당해보는 것이다

- 표현식인 문은 값으로 평가되므로 변수에 할당할 수 있다.

```js
var foo = var x;
```

- 표현식이 아닌 문은 값처럼 사용할 수 없다

```js
var x; // 변수 선언문 : 표현식이 아닌 문

x = 100; // 할당문 : 표현식이자 완전한 문
```

```js
var foo = (x = 100);
x = 100;
```

- 표현식인 문은 값처럼 사용할 수 있다
