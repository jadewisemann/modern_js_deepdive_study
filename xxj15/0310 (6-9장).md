### ⚡️ 원시 타입 : 불변

- 배정밀도 64비트 부동 소수점 -> 1이랑 1.0 동일하게 취급

  - `Infinity`: 양의 무한대
  - `-Infinity`: 음의 무한대

#### 템플릿 리터럴

- 탬플릿 리터럴은 백틱(``)을 사용해 표현한다.
- 일반 문자열 내에서는 줄바꿈 (개행)이 허용되지 않지만 템플릿 리터럴을 사용하면 이스케이프 시퀀스를 사용하지 않고도 줄바꿈이 허용되며, 모든 공백도 있는 그대로 적용된다.

- 표현식 삽입

```js
var first = "seojin";
var last = "An";

console.log(`My name is ${first} ${last}.`);
```

3. boolean

4. undefined : var 키워드로 선언한 변수는 암묵적으로 undefined로 초기화된다.
   변수를 참조했을 때 undefined가 반환된다면 참조한 변수가 선언 이후 값이 할당된 적 없는, 즉 초기화되지 않은 변수라는 것을 알 수 있다. (메모리 공간 잡아놓기 위해서 js가)

5. null : 변수에 값이 없는 경우 / 함수가 유효한 값을 반환할 수 없는 경우

### ⚡️ 객체 타입

- 원시타입 외의 모든 것

- 객체, 함수, 배열 등

### ⚡️ 데이터 타입의 필요성

- 값은 메모리에 저장하고 참조할 수 있어야 하는데, 메모리에 값을 저장하려면 먼저 확보해야 할 메모리 공간의 크기를 결정해야 한다.

- JS 엔진은 데이터 타입, 즉 값의 종류에 따라 정해진 크기의 메모리 공간을 확보하는데, 즉 변수에 할당되는 값의 데이터 타입에 따라 확보해야 할 메모리 공간의 크기가 정해진다.

#### 동적 타이핑

- C나 자바같은 정적 타입 언어는 변수를 선언할 때 변수에 할당할 수 있는 값의 종류, 즉 데이터 타입을 사전에 선언해야한다. (**명시적 타입 선언**)
  - 이런 언어는 변수의 타입을 변경할 수 없으며, 변수에 선언한 타입에 맞는 값만 할당 가능하다
- JS는 이와 달리 변수를 선언할 때 **타입을 선언하지 않는다.**
  - 다만, `var`, `let`, `const` 키워드를 사용해 변수를 선언할 뿐이다.
  - 따라서 어떠한 타입의 데이터 값이라도 자유롭게 할당할 수 있다.

💡 **동적 타이핑** : JS의 변수는 선언이 아닌 **할당**에 의해 타입이 결정(타입추론)되고, 재할당에 의해 변수의 타입은 언제든지 **동적으로 변할 수 있**다.

## 📍 7장 : 연산자

1. **산술 연산자**

- 이항 산술 연산자 (+, -, \*, /, %) : 두개의 피연산자를 산술 연산
- 단항 산술 연산자 (++, --, +, -) : 한개의 피연산자를 산술 연산
- 문자열 연결 연산자 (+) : 피연산자 중 하나 이상이 문자열인 경우 문자열 연결
  - **암묵적 타입 변환** / **타입 강제 변환** 이루어짐

2. **할당 연산자**

- =, +=, -= 등

3. **비교 연산자**

- ==, !=, >, < , <=. >= : 값 비교
- ===, !== : 타입 비교

- 또한 동등비교 연산자는
  - 비교할때 암묵적 타입 변환을 통해 타입을 일치 시킨후 비교
  - 일치 비교는 하지 않는다 => 예측 가능함

4. **삼항 조건 연산자**

5. **논리 연산자**

- || , && , !
  - 언제나 불리언 값을 반환하는 논리 부정 연산자와 달리 논리합(`||`) 또는 논리곱(`&&`) 연산자 표현식의 평가 결과는 불리언 값이 아닐 수도 있다.

  ```js
  // 단축 평가
  "cat" && "Dog"; // 'Dog'
  ```

6. **쉼표 연산자** : 왼쪽 피연산자부터 차례대로 평가하고 마지막 피연산자의 평가가 끝나면 마지막 피연산자의 평가 결과 반환

```js
var x, y, z;

((x = 1), (y = 2), (z = 3)); // 3
```

7. **그룹 연산자**
8. **typeof** 연산자

- 주의할 점 : 값이 `null` 타입인지 확인할때는 `typeof` 대신 `===` 사용할 것
  - typeof 로 null 값 연산하면 null이 아닌 **object**를 반환함.
  - 함수는 `function` 을 반환 (object임에도)

9. **지수 연산자** : `**`

## 📍 8장 : 제어문 (조건문과 반복문)

### 1. 블록문

- 0개 이상의 문을 중괄호로 묶은 것. **코드 블록** 또는 **블록** 이라고 부름
  - JS에서 블록문은 하나의 실행 단위.

```js
{
  var foo = 10;
}
```

- 문의 끝에 `;`를 붙이는게 일반적이지만 블록문은 언제나 문의 종료를 의미하는 **자체 종결성**을 갖기 때문에 세미콜론을 붙이지 않는다.

### 2. 조건문

- **if~else 문**

```js
var result = x % 2 ? "홀수" : "짝수";
```

이런 식으로 삼항 조건 연산자로 바꿔쓸 수 있다.

- **switch 문**

```js
var month = 11;
var monthName;

switch (month) {
  case 1: monthName = 'January';
  ...
  case 11: monthName = 'November';
  case 12: monthName = 'December';
  default : monthName = 'Invalid Month';
}
```

- **Fall through** : 위 switch문의 실행 결과 `'Invalid Month'`가 출력되는데, 이는 문을 실행한 후 switch문을 탈출할 수 있는 break문이 없었기 때문이다.
  - 다시 말해, monthName 변수에 'November'가 할당된 후 switch문을 탈출하지 않고 'December', 'Invalid Month'가 재할당되었다.

### 3. 반복문

- for문, while문, do~while문이 존재한다

#### ⚡️ for문

- 반복 횟수가 명확할 때 주로 사용

```js
for (var i = 0; i < 2; i++) {
  console.log(i);
}
```

#### ⚡️ while 문

- 반복 횟수가 명확하지 않을 때 주로 사용

```js
while (true) {
  // 무한루프

  if (cnt == 3) break; // 코드 블럭 탈출
}
```

#### ⚡️ do... while 문

- 코드 블록을 먼저 실행하고 조건식 평가

```js
do {
  // 코드 블럭은 무조건 한 번 이상 실행됨
} while (cnt < 3);
```

## 📍 9장 : 타입 변환과 단축 평가

### 9.1 타입 변환

- **명시적 타입 변환** 또는 **타입 캐스팅** : 개발자가 의도적으로 값의 타입을 변화시키는 것

```js
(10).toString();
```

- **암묵적 타입 변환** 또는 **타입 강제 변환** : 개발자의 의도와 상관없이 표현식을 평가하는 도중에 JS 엔진에 의해 암묵적으로 타입이 자동변화된 것

```js
10 + "";
```

### 9.2 암묵적 타입 변환

- 문자열 타입으로 변환
- 숫자 타입으로 변환
  - 숫자로 변화 시도했지만 안되면 `NaN` 반환
- 불리언 타입으로 변환
  - JS엔진은 불리언 타입이 아닌 값을 **Truthy 값(참으로 평가되는 값)** 또는 **Falsy 값(거짓으로 평가되는 값)**으로 구분한다. 따라서, 제어문의 조건식과 같이 불리언 값으로 평가되어야 할 문맥에서 Truthy 값은 `true`로, Falsy 값은 `false`로 암묵적 타입 변환된다.

  - Falsy 값
  - `false`, `undefined`, `null`, `0`, `-0`, `Nan`, `''`
  - 📍 **객체는 무조건 Truthy -> `[]` 비어있는 객체도 Truthy!!!**

### 9.3 명시적 타입 변환

#### 1. 문자열 타입으로 변환

```js
//1. String 생성자 함수
String(1);

// 2. Object.prototype.toString 메서드
(1).toString();

//3. 문자열 연결 연산자
1 + "";
```

#### 2. 숫자 타입으로 변환

```js
//1. Number 생성자 함수
Number("1");

// 2. parseInt, parseFloat 함수 - 문자열만 가능
parseInt("0");

//3. + 단항 산술 연산자
+"0";

//4. * 산술 연산자
"0" * 1;
```

#### Boolean 타입으로 변환

```js
// 1. Boolean 생성자 함수
Boolean(""); // false

// 2. @ 부정 논리 연산자 두 번 사용
!!"x";
```

### 9.4 단축 평가

- 논리곱(&&) 연산자

```js
"Cat" && "Dog"; // Dog
```

- 두 개의 피연산자가 모두 true로 평가될 때 true를 반환
  - 논리 연산의 결과를 결정하는 두 번째 피연산자, 즉 문자열 'Dog'를 그대로 반환한다

- 논리합 (||) 연산자

```js
"Cat" || "Dog"; // Cat
```

- 두 개의 피연산자 중 하나만 true로 평가되어도 true를 반환
  - 논리 연산의 결과를 결정한 첫 번째 피연산자, 즉 Catdㅡㄹ 그대로 반환한다

⚡️ **단축 평가**: 표현식을 평가하는 도중에 평가 결과가 확정된 경우 나머지 평가 과정을 생략하는 것

`true || anything` : true
`false || anything` : anything
`true && anything` : anything
`false && anything` : false

### 단축평가를 사용하면 유용한 경우

#### 1. 객체를 가리키기를 기대하는 변수가 null 또는 undefined가 아닌지 확인하고 프로퍼티를 참조할 때

```js
var elem = null;
var value = elem.value; // TypeError : Cannot read property 'value' of null
```

- 아래와 같이 작성하면 에러를 막을 수 있다

```js
var elem = null;
var value = elem && elem.value; // null
```

#### 2. 함수 매개변수에 기본값을 설정ㅇ할 때

```js
function getStringLength(str) {
  str = str || "";
  return str.length;
}
```

#### 옵셔널 체이닝 연산자

`?.` : 좌항의 피연산자가 null 또는 undefined인 경우 undefined 반환 / 그렇지 않으면 우항의 프로퍼티 참조

```js
var elem = null;

var value = elem?.value;
console.log(value); // undefined
```

#### null 병합 연산자

`??` : 좌항의 피연산자가 좌항의 피연산자가 `null` 또는 `undefined`인 경우 우항의 피연산자 반환 / 그렇지 않으면 좌항의 피연산자 반환

- 변수에 기본값을 설정할 때 유용함

```js
var foo = null ?? "default string";
console.log(foo);
```

- 좌항이 Truthy 하면 우항 절대 안나옴
