# 15 `let`, `const` 키워드와 블록 레벨 스코프

## 15.1`var` 키워드로 선언한 변수의 문제점

### 15.1.1 변수 중복 선언 허용
- 중복 선언이 가능함
	- 단, 초기화 없는 선언은 무시함
	- 값까지 할당하면서 초기화하면 그걸로 덮어 씌워짐
 
```js
var x = 1;
var y = 2;

var x; // 초기화 없음 => 무시됨
var y = 100; 덮어 씌워짐
```

### 15.1.2 함수 레벨 스코프  
- 블록 레벨 스코프
	- 오로지 함수의 코드 블록만 지역 스코프로 인정
	- 나머지는 모두 전역 스코프가 되어 버림

### 15.1.3 변수 호이스팅  
- 변수 호이스팅이 일어남
- 단, 할당 이전에는 언제나 `undefined`
	-  `undeifned`로 초기화 되기 때문

## 15.2 let 키워드

### 15.2.1 변수 중복 선언 금지
- 이름이 같으면 문법 에러 발생

### 15.2.2 블록 레벨 스코프  
- 모든 코드 블록을 지역 스코프로 인정

### 15.2.3 변수 호이스팅  
- `var`은 선언 후 즉시 초기화 됨(`undeiended`)
- `let`은 선언과 초기화를 나눠서 사용
	- `let`은 평가 단계에 선언되어 렉시컬 환경에 저장됨
	- 그리고 런타임에서 할당되면서 초기화 됨
	- 그 사이에 참조할 수 없는 순간
		- => *일시적 사각지대*
			- Temporal Dead Zone = TDZ

### 15.2.4 전역 객체와 let
- `var` 키워드의 전역 변수와 전역 함수 = 전역 객체 window의 프로퍼티
	- `window.x` 이런식으로 참조하는데 `window` 생략 가능

- `let`은 전역 객체의 프로퍼티가 아님
	- 렉시컬 환경의 레코드에 저장

## 15.3 const 키워드  
### 15.3.1 선언과 초기화  
- 반드시 선언과 동시에 초기화 해야 한다

### 15.3.2 재할당 금지  
- `var`, `let`은 재할당이 가능
- `const`는 재할당이 금지

### 15.3.3 상수  
- `const`에 원시값이 할당하면 변수 값을 변경 할 수 없다.
	- 상수로 사용 가능

- 상수는 재할당이 금지된 변수
	- `const` 키워드로 선언된 변수는 재할당 금지
	- 원시값은 변경 불가
- (따라서) 변경 할 수 있는 방법이 없음

### 15.3.4 const 키워드와 객체  
- `const`로 선언된 변수에 객체를 할당한 경우에는 값을 변경할 수 있음
-  재할당을 금지 할 뿐 불변을 이미하지는 않음
	- 상수로서의 역할은 불변한 원시 값이 행하는 것

## 15.4 var vs. let vs. const
- 보통 `const` 사용
- 재할당이 필요한 경우에만 `let`
